2//! # SBI (Supervisor Binary Interface) 封装
3//!
4//! 基于 `sbi-rt` 提供与 RISC-V SBI 固件的交互接口。该模块对常用功能
5//! 进行轻量封装，屏蔽底层调用细节，便于在内核中直接使用。
6//!
7//! ## 提供能力
8//! - 控制台 I/O：[`console_putchar`], [`console_getchar`]
9//! - 定时器：[`set_timer`]
10//! - 系统复位：[`shutdown`]
11//!
12//! ## 实现说明
13//! - 使用 `sbi_rt::legacy::*` 访问 legacy 扩展（兼容广泛）
14//! - `console_getchar` 的“无字符可读”返回值由固件实现决定（见函数文档）
15//! - 定时器使用绝对时间（timebase 计数）
16
17/// 控制台输出单个字符
18///
19/// 通过 SBI legacy 控制台接口输出一个字符。此调用为同步输出，
20/// 直至固件接受该字符。
21///
22/// ## Arguments
23/// * `c` - 要输出的字符（ASCII 码）
24pub fn console_putchar(c: usize) {
25    #[allow(deprecated)]
26    sbi_rt::legacy::console_putchar(c);
27}
28
29/// 控制台读取单个字符（非阻塞）
30///
31/// 从 SBI legacy 控制台尝试读取一个字符。
32/// - 有字符可读：返回其 ASCII 码
33/// - 无字符可读：返回值依赖固件实现，常见有两种：
34///   - 返回 `0`（本工程在 QEMU 环境中采用该语义）
35///   - 返回 `usize::MAX`（源自 legacy 规范中 `-1` 的无符号表示）
36///
37/// 建议调用方对两种情况都做兼容处理。
38pub fn console_getchar() -> usize {
39    #[allow(deprecated)]
40    sbi_rt::legacy::console_getchar()
41}
42
43/// 设置时钟中断触发时间（绝对时间）
44///
45/// 通过 SBI `set_timer` 配置下一次时钟中断触发的绝对时刻。
46/// 触发时刻以 timebase 计数为单位。
47///
48/// ## Arguments
49/// * `timer` - 触发时间（绝对计数值）
50pub fn set_timer(timer: usize) {
51    sbi_rt::set_timer(timer as _);
52}
53
54/// 关闭（或复位）系统
55///
56/// 使用 SBI System Reset 扩展请求系统关闭。根据 `failure` 参数选择
57/// 关闭原因码，以便固件或上层记录关机原因。
58///
59/// ## Arguments
60/// * `failure` - 是否因为系统故障而关闭
61///   - `false`：`NoReason`
62///   - `true`：`SystemFailure`
63///
64/// ## Behavior
65/// - 正常情况下，此函数不会返回。
66pub fn shutdown(failure: bool) -> ! {
67    use sbi_rt::{NoReason, Shutdown, SystemFailure, system_reset};
68    if !failure {
69        system_reset(Shutdown, NoReason);
70    } else {
71        system_reset(Shutdown, SystemFailure);
72    }
73    unreachable!()
74}
