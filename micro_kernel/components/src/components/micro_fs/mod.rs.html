1//! # Micro File System (Micro-FS)
2//!
3//! 一个简单的文件系统实现，
4//! 提供了完整的文件系统功能，包括文件管理、目录结构、块缓存等。
5//!
6//! ## 系统架构
7//!
8//! ```text
9//! ┌─────────────────────────────────────────────────────────────┐
10//! │                        Micro-FS                             │
11//! ├─────────────────────────────────────────────────────────────┤
12//! │    VFS Layer    │   MFS Layer   │        Block Layer        │
13//! │    (vfs.rs)     │   (mfs.rs)    │     (block_cache.rs)      │
14//! ├─────────────────────────────────────────────────────────────┤
15//! │     Layout      │    Bitmap     │        Block Device       │
16//! │   (layout.rs)   │  (bitmap.rs)  │      (block_dev.rs)       │
17//! └─────────────────────────────────────────────────────────────┘
18//! ```
19//!
20//! ## 核心模块
21//!
22//! - **VFS Layer**: 虚拟文件系统接口，提供统一的文件操作 API
23//! - **MFS Layer**: Micro File System 核心实现，管理文件系统元数据
24//! - **Layout**: 文件系统布局定义，包含超级块、inode、目录项等结构
25//! - **Bitmap**: 位图管理，用于跟踪数据块和 inode 的分配状态
26//! - **Block Cache**: 块缓存管理，提高 I/O 性能
27//! - **Block Device**: 块设备抽象接口
28//!
29//! ## 文件系统布局
30//!
31//! ```text
32//! ┌─────────────────────────────────────────────────────────────┐
33//! │                        SuperBlock                           │
34//! ├─────────────────────────────────────────────────────────────┤
35//! │                       Inode Bitmap                          │
36//! ├─────────────────────────────────────────────────────────────┤
37//! │                        Inode Area                           │
38//! ├─────────────────────────────────────────────────────────────┤
39//! │                        Data Bitmap                          │
40//! ├─────────────────────────────────────────────────────────────┤
41//! │                         Data Area                           │
42//! └─────────────────────────────────────────────────────────────┘
43//! ```
44//!
45//! ## 主要特性
46//!
47//! - **简单高效**: 设计简洁，易于理解和扩展
48//! - **类型安全**: 充分利用 Rust 的类型系统保证内存安全
49//! - **缓存优化**: 内置块缓存机制，提高 I/O 性能
50//! - **并发安全**: 支持多线程访问，使用锁机制保证数据一致性
51//! - **可移植性**: 抽象块设备接口，支持不同的存储介质
52//!
53//! ## 使用示例
54//!
55//! ```rust
56//! use micro_fs::{MicroFileSystem, BlockDevice};
57//!
58//! // 创建文件系统
59//! let mfs = MicroFileSystem::create(block_device, 4096, 1, 4096, 4096);
60//!
61//! // 打开根目录
62//! let root_inode = mfs.root_inode();
63//!
64//! // 创建文件
65//! let file = root_inode.create("test.txt").unwrap();
66//!
67//! // 写入数据
68//! file.write_at(0, b"Hello, Micro-FS!");
69//!
70//! // 读取数据
71//! let mut buf = [0u8; 16];
72//! file.read_at(0, &mut buf);
73//! ```
74//!
75//! ## 性能特点
76//!
77//! - **块大小**: 512 字节，平衡空间利用率和 I/O 效率
78//! - **缓存大小**: 16 个块，减少磁盘访问次数
79//! - **索引结构**: 三级索引，支持最大 8MB 文件
80//! - **分配策略**: 首次适应算法，快速定位可用资源
81//!
82//! ## 限制说明
83//!
84//! - 文件名最大长度：27 个字符
85//! - 单个文件最大大小：约 8MB
86//! - 文件系统最大大小：取决于块设备容量
87//! - 并发访问：支持多线程，但需要适当的锁管理

104/// 文件系统块大小（字节）
105///
106/// 每个块包含 512 字节，这是文件系统的基本存储单位。
107/// 块大小影响文件系统的空间利用率和 I/O 性能。
108///
109/// ## 选择考虑
110///
111/// - **空间利用率**: 较小的块大小减少内部碎片
112/// - **I/O 效率**: 较大的块大小提高顺序访问性能
113/// - **内存使用**: 块大小影响缓存的内存占用
114/// - **兼容性**: 512 字节是常见的块设备扇区大小
115pub const BLOCK_SZ: usize = 512;
