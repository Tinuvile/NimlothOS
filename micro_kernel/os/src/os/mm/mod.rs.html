1//! # 内存管理模块
2//!
3//! 提供完整的内存管理功能，包括地址管理、页表操作、物理页帧分配、
4//! 堆分配器和地址空间管理。支持 RISC-V SV39 分页机制。
5//!
6//! ## 模块组织
7//!
8//! - [`address`] - 地址和页号的类型安全封装，支持地址对齐和转换
9//! - [`frame_allocator`] - 物理页帧分配器，管理物理内存页面
10//! - [`heap_allocator`] - 内核堆分配器，支持动态内存分配
11//! - [`memory_set`] - 地址空间管理，支持内存映射和地址空间切换
12//! - [`page_table`] - 页表管理，实现虚拟地址到物理地址的转换
13//!
14//! ## 初始化流程
15//!
16//! 内存管理系统的初始化顺序：
17//! 1. **堆分配器初始化** - 启用内核动态内存分配
18//! 2. **页帧分配器初始化** - 设置物理内存管理
19//! 3. **内核地址空间激活** - 启用虚拟内存管理
20//!
21//! ## 核心类型
22//!
23//! ### 地址类型
24//! - [`PhysAddr`] / [`VirtAddr`] - 物理地址和虚拟地址
25//! - [`PhysPageNum`] / [`VirtPageNum`] - 物理页号和虚拟页号
26//!
27//! ### 页表管理
28//! - [`PageTable`] - 多级页表结构
29//! - [`PageTableEntry`] - 页表项
30//! - [`PTEFlags`] - 页表项标志位
31//!
32//! ### 内存分配
33//! - [`FrameTracker`] - 物理页帧的RAII管理
34//! - [`frame_alloc`] / [`frame_dealloc`] - 页帧分配和释放
35//!
36//! ### 地址空间管理
37//! - [`MemorySet`] - 完整的地址空间
38//! - [`MapPermission`] - 内存访问权限
39//! - [`KERNEL_SPACE`] - 全局内核地址空间
40//!
41//! ## 使用示例
42//!
43//! ```rust
44//! // 初始化内存管理系统
45//! mm::init();
46//!
47//! // 分配物理页帧
48//! let frame = frame_alloc().unwrap();
49//! let ppn = frame.ppn;
50//!
51//! // 地址转换
52//! let va = VirtAddr::from(0x10000000);
53//! let vpn = va.floor();
54//!
55//! // 访问内核地址空间
56//! let mut kernel_space = KERNEL_SPACE.exclusive_access();
57//! kernel_space.activate();
58//! ```

74/// 初始化内存管理系统
75///
76/// 按照正确的顺序初始化内存管理的各个组件，确保系统能够正常进行
77/// 内存分配和虚拟地址转换。
78///
79/// ## 初始化步骤
80///
81/// 1. **堆分配器初始化** - 调用 [`heap_allocator::init_heap()`]
82///    - 设置内核堆的起始地址和大小
83///    - 启用 `alloc` crate 的动态内存分配功能
84///    - 支持 `Vec`、`BTreeMap` 等集合类型
85///
86/// 2. **页帧分配器初始化** - 调用 [`frame_allocator::init_frame_allocator()`]
87///    - 计算可分配的物理内存范围
88///    - 初始化物理页帧的空闲列表
89///    - 设置页帧分配和回收机制
90///
91/// 3. **内核地址空间激活** - 调用 [`KERNEL_SPACE.exclusive_access().activate()`]
92///    - 创建内核地址空间的页表映射
93///    - 设置 `satp` 寄存器启用分页机制
94///    - 刷新 TLB 确保地址转换正确
95///
96/// ## 调用时机
97///
98/// 此函数应在内核初始化的早期阶段调用，通常在 BSS 段清零和日志系统
99/// 初始化之后，但在任何需要动态内存分配的操作之前。
100///
101/// ## Panics
102///
103/// 如果内存管理系统初始化失败（如物理内存不足、页表创建失败等），
104/// 函数会触发 panic，因为这是系统运行的基础要求。
105pub fn init() {
106    heap_allocator::init_heap();
107    frame_allocator::init_frame_allocator();
108    KERNEL_SPACE.exclusive_access().activate();
109}
