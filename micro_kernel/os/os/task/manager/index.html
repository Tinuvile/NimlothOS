§任务管理器模块

提供任务调度和管理的核心功能，实现就绪队列的维护和任务分发。 采用 FIFO (First In, First Out) 调度策略，确保任务调度的公平性和可预测性。

§核心组件

  • TaskManager - 任务管理器，维护就绪任务队列
  • TASK_MANAGER - 全局任务管理器实例
  • add_task - 向就绪队列添加任务的全局接口
  • fetch_task - 从就绪队列获取任务的全局接口

§设计原理

§调度策略

采用简单而高效的 先进先出 (FIFO) 调度算法：

  • 公平性：所有任务按照进入就绪队列的顺序被调度
  • 简洁性：避免复杂的优先级计算和时间片管理
  • 可预测性：调度行为完全可预测，便于调试和测试

§数据结构选择

使用 VecDeque<Arc<TaskControlBlock>> 作为就绪队列：

  • 双端队列：支持高效的队首删除和队尾插入操作
  • 引用计数：Arc 允许多个调度器组件共享任务控制块
  • 时间复杂度：入队和出队操作均为 O(1)

§并发安全

通过 UPSafeCell 实现单处理器环境下的线程安全：

  • 互斥访问：确保同一时间只有一个线程可以修改就绪队列
  • 内部可变性：允许在不可变引用下修改内部数据
  • 无锁设计：在单核环境下避免昂贵的锁操作

§性能特征

  • 入队延迟: O(1) - 常数时间复杂度
  • 出队延迟: O(1) - 常数时间复杂度
  • 内存开销: 每个任务一个 Arc 指针 (8 bytes)
  • 缓存友好性: 连续内存布局提供良好的缓存局部性

§使用示例

use crate::task::manager::{add_task, fetch_task};
use crate::task::task::TaskControlBlock;
use alloc::sync::Arc;

// 添加新任务到就绪队列
let new_task = Arc::new(TaskControlBlock::new(app_data));
add_task(new_task);

// 调度器获取下一个任务
if let Some(task) = fetch_task() {
    println!("调度任务 PID: {}", task.getpid());
    // 执行任务...
} else {
    println!("没有就绪任务，进入空闲状态");
}
