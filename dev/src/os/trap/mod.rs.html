1//! # 陷阱和中断处理模块
2//!
3//! 处理所有从用户态到内核态的陷阱 (trap)，包括系统调用、异常和中断。
4//! 陷阱是 RISC-V 架构中用户态与内核态交互的核心机制。
5//!
6//! ## 处理的陷阱类型
7//!
8//! - **系统调用** (`UserEnvCall`): 用户程序请求内核服务
9//! - **时钟中断** (`SupervisorTimer`): 实现抢占式多任务调度
10//! - **页面异常** (`StoreFault`, `StorePageFault`): 内存访问违规
11//! - **非法指令** (`IllegalInstruction`): 执行无效指令
12//!
13//! ## 执行流程
14//!
15//! 1. **陷阱触发**: 用户态程序执行 `ecall` 或发生异常/中断
16//! 2. **硬件切换**: CPU 自动切换到 S 模式，跳转到 `stvec` 指定的处理程序
17//! 3. **上下文保存**: `__alltraps` 保存所有寄存器到陷阱上下文
18//! 4. **处理分发**: `trap_handler` 根据陷阱类型执行相应处理
19//! 5. **上下文恢复**: `__restore` 恢复寄存器并返回用户态
20//!
21//! ## 寄存器使用
22//!
23//! - `stvec`: 陷阱向量寄存器，指向陷阱处理入口 `__alltraps`
24//! - `scause`: 陷阱原因寄存器，标识陷阱类型和具体原因
25//! - `stval`: 陷阱值寄存器，包含相关的地址或值信息
26//! - `sstatus`: 状态寄存器，控制中断使能和特权级
27//! - `sepc`: 异常程序计数器，指向触发陷阱的指令地址
28

45/// 初始化陷阱处理系统
46///
47/// 设置陷阱向量寄存器 `stvec`，指向陷阱处理入口点 `__alltraps`。
48/// 必须在系统启动早期调用，在任何可能触发陷阱的操作之前。
49///
50/// ## 配置内容
51///
52/// - 将 `stvec` 设置为 `__alltraps` 函数地址
53/// - 使用直接模式 (`TrapMode::Direct`)，所有陷阱都跳转到同一个处理程序
54///
55/// ## Safety
56///
57/// 此函数是安全的，但内部使用 `unsafe` 访问 CSR 寄存器。
58pub fn init() {
59    unsafe extern "C" {
60        fn __alltraps();
61    }
62    unsafe {
63        stvec::write(__alltraps as usize, TrapMode::Direct);
64    }
65}

67/// 启用时钟中断
68///
69/// 在监督者中断使能寄存器 (`sie`) 中启用时钟中断位，
70/// 允许时钟中断触发陷阱进入内核进行任务调度。
71///
72/// ## 功能
73///
74/// - 设置 `sie.STIE` 位，启用监督者时钟中断
75/// - 配合时钟设备实现抢占式多任务调度
76/// - 必须在时钟设备配置完成后调用
77///
78/// ## Safety
79///
80/// 此函数是安全的，但内部使用 `unsafe` 访问 CSR 寄存器。
81pub fn enable_timer_interrupt() {
82    unsafe {
83        sie::set_stimer();
84    }
85}

87/// 陷阱处理器主函数
88///
89/// 这是所有陷阱的统一处理入口，由汇编代码 `__alltraps` 调用。
90/// 根据陷阱类型分发到不同的处理逻辑，处理完成后返回修改后的陷阱上下文。
91///
92/// ## Arguments
93///
94/// * `cx` - 陷阱上下文的可变引用，包含触发陷阱时的 CPU 状态
95///
96/// ## Returns
97///
98/// 返回修改后的陷阱上下文，供 `__restore` 恢复到用户态
99///
100/// ## 处理的陷阱类型
101///
102/// ### 1. 系统调用 (`UserEnvCall`)
103///
104/// - 将 `sepc` 加 4，跳过 `ecall` 指令
105/// - 从寄存器提取系统调用号和参数
106/// - 调用 [`syscall`] 执行具体的系统调用
107/// - 将返回值写入 `a0` 寄存器 (`cx.x[10]`)
108///
109/// ### 2. 存储异常 (`StoreFault`, `StorePageFault`)
110///
111/// - 记录故障地址 (`stval`) 和指令地址 (`sepc`)
112/// - 输出错误信息到内核日志
113/// - 终止当前任务并调度下一个任务
114///
115/// ### 3. 非法指令 (`IllegalInstruction`)
116///
117/// - 记录异常指令地址 (`sepc`)
118/// - 输出错误信息到内核日志  
119/// - 终止当前任务并调度下一个任务
120///
121/// ### 4. 监督者时钟中断 (`SupervisorTimer`)
122///
123/// - 设置下一次时钟中断触发时间
124/// - 挂起当前任务并切换到下一个就绪任务
125/// - 实现抢占式多任务调度
126///
127/// ## 寄存器约定
128///
129/// 遵循 RISC-V ABI 约定：
130/// - `x10` (`a0`): 系统调用返回值 / 第一个参数
131/// - `x11` (`a1`): 系统调用第二个参数
132/// - `x12` (`a2`): 系统调用第三个参数  
133/// - `x17` (`a7`): 系统调用号
134///
135/// ## 执行流程
136///
137/// ```text
138/// 陷阱触发 -> __alltraps -> trap_handler -> __restore -> 用户态
139/// ```
140///
141/// ## Safety
142///
143/// - 使用 `#[unsafe(no_mangle)]` 确保函数名不被修改，供汇编代码调用
144/// - 函数本身是安全的，所有 unsafe 操作都在子函数中处理
145///
146/// ## Panics
147///
148/// 遇到不支持的陷阱类型时会触发 panic，这通常表示：
149/// - 硬件故障
150/// - 用户程序触发了预期外的异常
151/// - 内核配置错误
152#[unsafe(no_mangle)]
153pub fn trap_handler(cx: &mut TrapContext) -> &mut TrapContext {
154    let scause = scause::read();
155    let stval = stval::read();
156    match scause.cause() {
157        Trap::Exception(Exception::UserEnvCall) => {
158            // 系统调用处理
159            cx.sepc += 4; // 跳过 ecall 指令
160            cx.x[10] = syscall(cx.x[17], [cx.x[10], cx.x[11], cx.x[12]]) as usize;
161        }
162        Trap::Exception(Exception::StoreFault) | Trap::Exception(Exception::StorePageFault) => {
163            // 存储异常：非法内存访问
164            println!(
165                "[kernel] PageFault in application, bad addr = {:#x}, bad instruction = {:#x}, kernel killed it.",
166                stval, cx.sepc
167            );
168            exit_current_and_run_next();
169        }
170        Trap::Exception(Exception::IllegalInstruction) => {
171            // 非法指令异常：执行了无效的指令
172            println!(
173                "[kernel] IllegalInstruction in application, bad instruction = {:#x}, kernel killed it.",
174                cx.sepc
175            );
176            exit_current_and_run_next();
177        }
178        Trap::Interrupt(Interrupt::SupervisorTimer) => {
179            // 时钟中断：实现抢占式调度
180            set_next_trigger();
181            suspend_current_and_run_next();
182        }
183        _ => {
184            // 未处理的陷阱类型
185            panic!(
186                "Unsupported trap {:?}, stval = {:#x}!",
187                scause.cause(),
188                stval,
189            );
190        }
191    }
192    cx
193}
