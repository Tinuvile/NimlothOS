pub fn kernel_stack_position(app_id: usize) -> (usize, usize)
Expand description

计算指定应用程序的内核栈位置

每个应用程序在虚拟地址空间中都有独立的内核栈，用于处理该应用程序 的系统调用和中断。内核栈从高地址向低地址增长，位于跳板页之下。

§Arguments

  • app_id - 应用程序 ID，从 0 开始编号

§Returns

返回一个元组 (bottom, top)，表示该应用程序内核栈的地址范围：

  • bottom - 栈底地址（低地址）
  • top - 栈顶地址（高地址）

         |-- PAGE_SIZE (GUARD PAGE) --|
