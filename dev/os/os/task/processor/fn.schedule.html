Function schedule

Source 
pub fn schedule(switched_task_cx_ptr: *mut TaskContext)
Expand description

任务调度函数 - 从当前任务切换回调度器

将执行控制权从当前正在运行的任务转移回调度器的核心函数。 通常在时间片到期、任务主动让出 CPU 或任务阻塞时调用。

§Arguments

  • switched_task_cx_ptr - 指向当前任务上下文的可变指针，用于保存任务状态

§调度时机

§抢占式调度

// 时钟中断处理中
pub fn handle_timer_interrupt() {
    // 处理时间片到期
    set_next_trigger();
     
    // 切换到调度器
    schedule(current_task_cx_ptr);
}

§协作式调度

// yield 系统调用
pub fn sys_yield() -> isize {
    // 任务主动让出 CPU
    schedule(current_task_cx_ptr);
    0
}

§阻塞调度

// 任务等待资源时
pub fn sys_wait_for_resource() -> isize {
    // 将任务加入等待队列
    add_to_wait_queue(current_task());
     
    // 切换到其他任务
    schedule(current_task_cx_ptr);
}

§调用约定

§调用前要求

  • 当前任务的上下文必须已经保存到 switched_task_cx_ptr 指向的位置
  • 任务状态应该已经适当更新（Ready, Blocked 等）
  • 如需要，任务应该已经加入相应的队列（就绪队列、等待队列等）

§调用后保证

  • 控制权转移到调度器
  • 调度器会选择下一个任务执行
  • 当前任务可能稍后被重新调度

§性能考虑

  • 切换开销: 约 100-200 CPU 周期
  • 内存访问: 主要是寄存器保存/恢复
  • 缓存影响: 可能导致缓存未命中
  • TLB 影响: 如果切换到不同进程，可能导致 TLB 刷新

§使用示例

§时间片轮转

// 在时钟中断处理函数中
pub extern "C" fn timer_interrupt_handler() {
    // 更新时间片计数...
     
    // 时间片到期，调度其他任务
    let current_cx_ptr = current_task_context_ptr();
     
    // 将当前任务重新加入就绪队列
    if let Some(task) = take_current_task() {
        add_task(task);
    }
     
    // 切换到调度器选择新任务
    schedule(current_cx_ptr);
}

§安全性考虑

  • 使用 unsafe 代码进行底层上下文切换
  • 必须确保传入的指针有效且指向正确的上下文结构
  • 调用时必须在适当的内核态上下文中

§与 run_tasks() 的协作

schedule() 和 run_tasks() 构成完整的调度循环：

  • run_tasks(): 从调度器切换到任务
  • schedule(): 从任务切换回调度器
