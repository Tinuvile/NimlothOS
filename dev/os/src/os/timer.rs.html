1//! # 时钟和定时器管理模块
2//!
3//! 提供系统时间读取和定时器管理功能，支持抢占式多任务调度的时间片机制。
4//! 基于 RISC-V 的 `time` CSR 寄存器和 SBI 时钟接口实现。
5//!
6//! ## 核心功能
7//!
8//! - **时间读取**: 获取系统启动以来的时钟周期数和毫秒数
9//! - **定时器设置**: 配置下一次时钟中断的触发时间
10//! - **时间片调度**: 支持基于时间片的抢占式任务调度
11//!
12//! ## 时钟配置
13//!
14//! - **时钟频率**: 由 [`CLOCK_FREQ`] 定义，通常为 12.5MHz (QEMU 环境)
15//! - **时间片频率**: 100Hz，即每 10ms 触发一次时钟中断
16//! - **时间精度**: 毫秒级时间测量精度
17//!
18//! ## 使用场景
19//!
20//! - 任务调度的时间片管理
21//! - 系统调用 `get_time()` 的实现
22//! - 性能测量和基准测试
23//! - 超时和延迟功能
24
25use crate::config::CLOCK_FREQ;
26use crate::sbi::timer;
27use riscv::register::time;
28
29/// 每秒的时钟中断次数 (100Hz)
30///
31/// 定义抢占式调度的时间片频率。100Hz 意味着每 10 毫秒触发一次
32/// 时钟中断，提供良好的响应性和合理的调度开销。
33const TICKS_PER_SEC: usize = 100;
34
35/// 每秒的毫秒数常量
36const MSEC_PER_SEC: usize = 1000;
37
38/// 获取当前系统时间（时钟周期数）
39///
40/// 返回系统启动以来的时钟周期数，用于时间测量和定时器功能。
41/// 时钟频率由 `CLOCK_FREQ` 常量定义。
42///
43/// ## Returns
44///
45/// 系统启动以来的时钟周期数
46///
47/// ## Examples
48///
49/// ```
50/// let start = time();
51/// // ... 执行一些操作 ...
52/// let end = time();
53/// let elapsed = end - start;
54/// ```
55pub fn time() -> usize {
56    riscv::register::time::read()
57}
58
59/// 获取当前系统时间（毫秒）
60///
61/// 将时钟周期数转换为毫秒，便于人类理解的时间表示。
62/// 转换公式：毫秒 = 时钟周期数 / (时钟频率 / 1000)
63///
64/// ## Returns
65///
66/// 系统启动以来的毫秒数
67///
68/// ## Examples
69///
70/// ```
71/// let start_ms = time_ms();
72/// // ... 执行一些操作 ...
73/// let end_ms = time_ms();
74/// let elapsed_ms = end_ms - start_ms;
75/// ```
76pub fn time_ms() -> usize {
77    time() / (CLOCK_FREQ / 1000)
78}
79
80/// 设置下一次时钟中断触发时间
81///
82/// 配置时钟设备在当前时间基础上加一个时间片间隔后触发中断，
83/// 实现定期的抢占式任务调度。
84///
85/// ## 工作原理
86///
87/// 1. 获取当前时间 (`time()`)
88/// 2. 计算下次中断时间：当前时间 + 时间片间隔
89/// 3. 通过 SBI 接口设置时钟中断触发时间
90/// 4. 当时钟到达设定时间时，硬件触发时钟中断
91///
92/// ## 时间片计算
93///
94/// 时间片间隔 = `CLOCK_FREQ / TICKS_PER_SEC`
95/// - 在 12.5MHz 时钟下，间隔为 125,000 个时钟周期
96/// - 对应 10 毫秒的时间片长度
97///
98/// ## Usage
99///
100/// 该函数通常在以下时机调用：
101/// - 系统初始化时设置第一次时钟中断
102/// - 时钟中断处理程序中设置下一次中断
103///
104/// ## Note
105///
106/// 必须在每次时钟中断处理时调用此函数，否则时钟中断将不会再次触发。
107pub fn next_trigger() {
108    timer(time() + CLOCK_FREQ / TICKS_PER_SEC);
109}
